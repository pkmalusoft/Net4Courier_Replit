# Shipment Meta Dashboard with User Notes – Functional Summary

## 1. Objective

Provide users with a **single shipment dashboard** that displays **all shipment meta details in a structured, easy-to-read format** and allows **simple internal user notes** for collaboration.

The focus is **quick reference**, **clarity**, and **low complexity**.

---

## 2. Shipment Dashboard – Overview

The dashboard is a **read-only snapshot** of shipment meta data with **sectioned grouping**.

### 2.1 Dashboard Structure

```
Shipment Header
Status & Alerts
Meta Details (Grouped Sections)
Manifest / MAWB Summary
User Notes
```

---

## 3. Shipment Header (Top Section)

Always visible.

* AWB Number
* Shipment Type (Document / Parcel)
* Mode of Shipment (Air / Sea / Land)
* Current Shipment Status
* Priority Indicator (if any)

Purpose: **Immediate identification**

---

## 4. Status & Alerts Section

* Condensed shipment status timeline
* On-Hold indicator (highlighted)
* Exception indicator (if any)

Purpose: **Instant operational awareness**

---

## 5. Shipment Meta Details (Grouped View)

Meta details are shown in **clearly separated sections** for easy scanning.

### 5.1 Customer & Parties

* Customer Name
* Account Type (Credit / Prepaid / COD)
* Consignor & Consignee names
* Billing Party

### 5.2 Routing Information

* Origin (City / Hub)
* Destination (City / Hub)
* Zone / Lane
* Mode of Transport

### 5.3 Shipment & Cargo

* Number of Pieces
* Weight (Actual / Chargeable)
* Declared Value
* Package Type
* Content Description

### 5.4 Compliance & Control

* On-Hold Status
* Hold Reason (if any)
* KYC / Customs flag

### 5.5 Operational Info

* Pickup Type
* SLA Type
* Expected Delivery Date

Purpose: **One-screen reference without deep navigation**

---

## 6. Manifest / MAWB Summary (If Applicable)

Displayed only if shipment is bagged or manifested.

* MAWB Number
* Bag Number
* Mode (Air / Sea / Land)
* Flight / Trip Number
* Origin Port → Destination Port

Purpose: **Linehaul visibility**

---

## 7. Financial Snapshot (Read-Only)

* Freight Charges
* Special Charges
* COD Amount
* Invoice Status
* Payment Status

Purpose: **Quick financial awareness**

---

## 8. User Comment Box (Simple Notes)

### 8.1 Purpose

Allow users to add **simple internal notes** for other users viewing the shipment.

### 8.2 Features

* Single text input box
* Add short comments (no categories, no attachments)
* Comments are visible to all internal users
* Newest comment appears first

### 8.3 @User Mention (Basic)

* Users can type `@Username`
* System highlights mentioned name
* Mentioned users receive a basic notification

> No complex tagging, workflows, or permissions.

---

## 9. User Comment Rules

* Notes are **informational only**
* Notes do **not** change shipment status
* Notes do **not** override On-Hold restrictions
* Notes remain visible even after delivery

---

## 10. Role-Based Behaviour (Simple)

* All operational users: View & Add notes
* Admin: Can delete inappropriate notes
* No editing after save (optional)

---

## 11. Performance & Simplicity Principles

* Meta details loaded from shipment master data
* Notes stored as lightweight text records
* No dependency on shipment event tables
* No complex joins or heavy queries

---

## 12. One-Line Business Rule

> *The shipment dashboard is a quick-reference control panel, and the user comment box is a lightweight internal communication aid.*

---

## 13. Implementation Scope (Clear Boundary)

Included:

* Meta detail dashboard
* Simple user notes with @mentions

Excluded:

* Advanced comment workflows
* Attachments
* External notifications
* Customer visibility
